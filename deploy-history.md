## Issues Encountered & Fixes

### Issue 1: Render Blueprint `type: static` not supported
- **Error**: `unknown type "static"` in render.yaml
- **Fix**: Changed frontend from `type: static` to `type: web` using `serve` package to host the built Angular SPA
- **Commit**: `aeef10b` - "Fix render.yaml: change frontend from static to web type"

### Issue 2: IPv6 ENETUNREACH connecting to Supabase
- **Error**: `connect ENETUNREACH 2406:da14:18fe:3100:6e54:cec4:85bb:deca:5432`
- **Root Cause**: Render free tier doesn't support IPv6 outbound. Supabase's direct hostname `db.xxxxx.supabase.co` resolves to IPv6 first.
- **Fix 1**: Added `--dns-result-order=ipv4first` to Node.js start command
- **Fix 2**: Switched to Supabase transaction pooler (`aws-0-ap-northeast-1.pooler.supabase.com:6543`) which uses IPv4
- **Fix 3**: Updated `database.js` to auto-encode special characters in passwords
- **Commits**: `9656eee`, `53560f9`, `66b4fb0`

### Issue 3: URI malformed error with DATABASE_URL
- **Error**: `URIError: URI malformed` at `decodeURIComponent`
- **Root Cause**: Password `K@Dst8@T%ffaJP-` contains `@` and `%` which break URL parsing
- **Fix**: Added `buildConnectionString()` function in `database.js` that auto-encodes special characters using `encodeURIComponent()`
- **Commit**: `66b4fb0` - "Fix: auto-encode special chars in DB password for connection string"

### Issue 4: Frontend build failed - global npm install blocked
- **Error**: `npm error code ENOENT` when running `npm install -g serve`
- **Root Cause**: Render blocks global npm installs
- **Fix**: Installed `serve` locally (`npm install serve`) and used `npx serve` instead
- **Commit**: `e57d435` - "Fix: install serve locally instead of globally"

### Issue 5: Tables not found in Supabase
- **Error**: `relation "medicine_records" does not exist`
- **Root Cause**: init.sql hadn't been run on Supabase
- **Fix**: Connected directly to Supabase and ran `init.sql` + seed script
- **Tables Created**: users, family_profiles, health_records, medicine_records, symptom_queries, response_logs, community_schemes, blood_banks, vaccination_camps, outbreak_alerts, symptom_cache

## Render Blueprint (`render.yaml`)

```yaml
services:
  # ---- Backend API ----
  - type: web
    name: arogya-sathi-backend
    runtime: node
    plan: free
    buildCommand: cd backend && npm install --omit=dev
    startCommand: cd backend && node --dns-result-order=ipv4first src/index.js
    envVars:
      - key: NODE_ENV
        value: production
      - key: DB_HOST
        sync: false
      - key: DB_PORT
        value: "5432"
      - key: DB_USER
        sync: false
      - key: DB_PASSWORD
        sync: false
      - key: DB_NAME
        value: "postgres"
      - key: DB_SSL
        value: "true"
      - key: JWT_SECRET
        generateValue: true
      - key: JWT_EXPIRES_IN
        value: 7d
      - key: LOG_LEVEL
        value: info
      - key: GEMINI_API_KEY
        sync: false
      - key: GROQ_API_KEY
        sync: false
    healthCheckPath: /api/health

  # ---- Frontend (Web Service serving built Angular SPA) ----
  - type: web
    name: arogya-sathi-frontend
    runtime: node
    plan: free
    buildCommand: >-
      cd frontend &&
      npm install &&
      sed -i "s|__API_URL__|$API_URL|g" src/environments/environment.prod.ts &&
      npm install serve &&
      npx ng build --configuration production
    startCommand: cd frontend && npx serve -s dist/arogya-sathi/browser -l 4200
    envVars:
      - key: API_URL
        sync: false
```

## Deployment Steps (Reproducible)

1. Push code to GitHub repo
2. Go to render.com -> New -> Blueprint -> Connect repo
3. Render auto-creates both services from `render.yaml`
4. Set environment variables on Render dashboard:
   - Backend: `DATABASE_URL`, `GEMINI_API_KEY`, `GROQ_API_KEY`
   - Frontend: `API_URL` = `https://arogya-sathi-backend.onrender.com/api`
5. Create tables in Supabase SQL Editor (paste init.sql)
6. Seed demo data
7. Deploy frontend with correct API_URL
8. Access at https://arogya-sathi-frontend.onrender.com

## Known Limitations

- **Backend free tier spins down** after 15 min of inactivity (~30s cold start)
- **Frontend free tier spins down** similarly
- First request after idle will be slow
- No Redis caching (not implemented in backend code yet)
